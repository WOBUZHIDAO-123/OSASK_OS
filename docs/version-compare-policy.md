# 版本对比策略

> 当将 qiu 版本与《30天自制操作系统》原版代码进行对比时，目标是与原版**几乎一模一样**。

## 核心原则：两个例外

**"几乎"的定义**——以下两种情况允许与原文不一致：

### 例外1：等效逻辑不改

逻辑完全相同，仅参数数值不同的代码（尤其是 UI/显示类），**保留 qiu 版本**，不强制对齐原版。

*判定标准：*
- 不会引起 bug
- 纯显示效果差异（坐标、颜色、字符串内容等）
- 变量/常量命名不同但语义等价

*示例：* 按钮位置坐标、字符串内容、窗口大小参数等，不改。

### 例外2：注释完全按 qiu 版

- **注释保留 qiu 版原样**，原版注释仅供参考。
- 如果发现原版注释比 qiu 版更好（更准确、更清晰），**提示用户**，由用户决定是否采纳。

## 例外之外的规则

凡不属于以上两个例外的差异——**都必须改到与原版一致**，包括但不限于：

- 拼写错误
- 缺少的函数/声明
- 错误的参数类型或数量
- 链接规则
- 逻辑流程差异

## 操作流程

1. 逐文件对比，识别差异
2. 判断是否属于两个例外 → 是则跳过（或提示用户）
3. 不属于例外 → 修改到与原版一致
4. 编译验证
