# GDT（全局描述符表）学习笔记

## 实模式 vs 保护模式的段

### 实模式（16 位）
- 寄存器只有 **16 位**，最大表示 64KB
- 为了访问 1MB 内存，采用 **段×16 + 偏移** 的拼接方式扩地址
- 段寄存器存的是**段基址**（如 `0x1000` → 左移4位 = `0x10000`）
- **没有权限检查**

```asm
MOV AX, 0x1000    ; 段基址 = 0x1000
MOV DS, AX
MOV [0x1234], AL  ; 偏移 = 0x1234
; 物理地址 = 0x1000 × 16 + 0x1234 = 0x11234
```

### 保护模式（32 位）
- 寄存器有 **32 位**，可直接表达 0～4GB
- 段不再用于扩地址，而是用于**权限管理**
- 段寄存器存的是**选择子（编号）**，不是段基址
- CPU 拿编号查 **GDT**，拿到基址+权限后才允许访问
- **带权限检查**（可读/可写/可执行/特权级）

```asm
MOV AX, 1*8       ; 选择子 = 8（指向 GDT[1]）
MOV DS, AX
MOV [0x1234], AL  ; 偏移 = 0x1234
; CPU 查 GDT[1] → 基址 = 0
; 物理地址 = 0 + 0x1234 = 0x1234
```

## 为什么需要 GDT？

保护模式下 CPU 不能直接信任任何地址，必须经过"门禁系统"：

```
你给的地址（偏移）
     ↓
CPU 查选择子 → 去 GDT 找对应的段描述符
     ↓
拿到段基址 + 段大小 + 权限
     ↓
检查权限（可读？可写？可执行？）
     ↓
物理地址 = 段基址 + 偏移
     ↓
允许访问 ✅ 或 抛出异常 ❌
```

## GDT 的内存布局

```
内存地址        内容
───────────────────────────────────
0x0026f800  ← IDT 起始（2KB，256个门描述符）
0x00270000  ← IDT 终点 / GDT 起始（64KB，8192个段描述符）
0x00280000  ← GDT 终点 / bootpack 加载地址
```

## 当前项目的 GDT 设置

| 槽位 | 基址 | 大小 | 权限 | ar | 用途 |
|------|------|------|------|-----|------|
| **[0]** | — | — | — | 0 | CPU 强制空描述符 |
| **[1]** | `0x00000000` | 4GB | 可读写 | `0x4092` | 系统整体段，访问全部内存 |
| **[2]** | `0x00280000` | 512KB | 可执行 | `0x409a` | 内核代码段，执行 bootpack |

## 选择子（Selector）结构

选择子占 16 位：

```
bit0-1 : RPL（请求特权级）— 0=内核级，3=用户级
bit2   : TI（表指示符）— 0=GDT，1=LDT
bit3-15: Index（在 GDT 中的槽位号）
```

`1×8` = `0x0008`：Index=1，TI=0（查GDT），RPL=0（内核级）

## ar（Access Rights）编码

```
0x4092 = 0100 0000 1001 0010 (二进制)
         G  D  L AVL | P DPL S TYPE
         1  1  0  0  | 1  00 1 0010
         │  │        │  │   │ └─ 可读写 (W=1)
         │  │        │  │   └─── 代码/数据段 (S=1)
         │  │        │   └────── ring0 内核级 (DPL=0)
         │  │        └────────── 存在位 (P=1)
         │  └─────────────────── 32位 (D=1)
         └────────────────────── 4KB粒度 (G=1)
```

## 关键理解

1. **GDT 不是"切蛋糕"（分区）**，而是给同一个物理地址贴不同的权限标签
2. **多个段可以重叠** — GDT[1] 覆盖全部 4GB，GDT[2] 覆盖 512KB，互不冲突
3. **偏移是段内偏移** — 保护模式下你给的地址始终是"从段开头算起"，CPU 自动加段基址
4. **GDT[1] 基址=0 是为了方便** — 这样偏移 = 物理地址，不用重新算地址
5. **实模式和保护模式写法相同但结果不同** — 同样的 `MOV [0x1234], AL`，CPU 用了两套完全不同的翻译规则

## 一句话总结

> 实模式分段是为了**扩地址**（16位→20位），保护模式分段是为了**查权限**。
> 段寄存器从"段基址"变成了"选择子编号"，CPU 拿到编号后查 GDT 确定权限。
