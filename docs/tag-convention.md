# Git 标签命名规范

> 从 **day07d** 之后开始执行此规范。
> 历史标签（day7、day07、day06、day05、day07c、day7b 等）保持不动，不修改、不删除。

---

## 格式

```
v{天数}{可选后缀}
```

### 规则

| 部分 | 约定 | 示例 |
|------|------|------|
| **前缀** | 统一使用小写 `v` | `v` |
| **天数** | 2 位数字（`01`~`99`） | `v07`、`v08`、`v10` |
| **后缀** | 小写字母（`b`、`c`、`d`…），表示当天第几次迭代 | `v07b`、`v08c` |

### 示例

| 标签 | 含义 |
|------|------|
| `v07` | 第 7 天第一个版本 |
| `v07b` | 第 7 天第二次提交 |
| `v07c` | 第 7 天第三次提交 |
| `v08` | 第 8 天第一个版本 |
| `v10` | 第 10 天第一个版本 |

### 注意事项

1. **天数必须 2 位** — `v07` 而非 `v7`，保证 GitHub 按字母序排列时 `v07`、`v08`、`v09`、`v10` 顺序正确
2. **后缀从 `b` 开始** — 当天第一个版本不带后缀，后续迭代按字母序递增
3. **全小写** — `v07D` 或 `V07d` 都不规范
4. **不含功能描述** — 功能描述写在 tag message 中，标签名只表版本

### 提交流程

1. 用户提供**提交信息**和**标签名**（按上述格式）
2. AI 执行 `git add` + `git commit -m "提交信息"`
3. AI 执行 `git tag -a {标签名} -m "提交信息"`
4. AI 执行 `git push && git push origin {标签名}`
